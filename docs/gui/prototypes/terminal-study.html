FOCUSED

The terminal owns the keyboard. Solid accent border + glow, active tab underlined in accent, a solid block cursor, and the status line reads keyboard. Every keystroke goes to the PTY; workspace hotkeys do not fire. Entered by clicking the terminal or Ctrl+`; the emulator is told it is focused (\x1b[I) — on terminal focus, not window focus.

VISIBLE · UNFOCUSED

Visible, but the editor owns the keyboard. The state that does not exist today — the whole point of the focus authority. Subtle border, content gently dimmed, a hollow cursor, status reads idle. You can read output and scrollback while s / b / v / m / x and the marking menu still drive the board. Opening the dock to glance no longer hijacks every keystroke.

DETACHED

A real focus state, not dead keys. Today a detached terminal silently swallows every keystroke. Here the process keeps running (dashed border, amber DETACHED badge), the screen dims behind an explicit Reattach affordance, and typing gives feedback instead of vanishing. Detached is a defined node in the focus authority, not an accidental fall-through.

Not in scope for this mock: the cell-grid rewrite, Unicode width, reflow, and damage tracking are emulator correctness — proved in code against the vttest / esctest2 conformance suites, not simulated here. This study settles only what needs the owner's eye: the ANSI palette against the locked identity, and the three visible focus affordances (decision 024 · TE-005).
